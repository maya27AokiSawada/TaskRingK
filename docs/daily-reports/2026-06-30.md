## 技術判断

### 日時型

ドメインモデルでは `java.time.Instant` を採用した。
理由:

- minSdk 26 で利用可能
- 不変でテストしやすい
- Firestore `Timestamp` / Room epochMillis との変換を Mapper に集約しやすい

### Room の位置付け

Room は Source of Truth ではなく、Firestore のローカルキャッシュとして扱う。
そのため、マイグレーションは将来的にも破棄・再同期可能な設計を基本とする。

### JSON 保存

`SharedList.items` と `SharedGroup.members` は Room に正規化せず JSON 文字列で保存する方針にした。
理由:

- Phase 1 時点ではローカル表示・キャッシュ用途が主目的
- Firestore 側の構造（`items` は Map）との対応が取りやすい
- 差分更新の責務は Firestore DataSource / Repository 側に置くため
