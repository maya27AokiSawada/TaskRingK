## 技術判断

### Flow エラーのフォールバック

`flow { throw ... }` で生成した Flow でないと `.catch` が機能しない。`every { mock.fn() } throws RuntimeException` は呼び出し時に同期的にスローするため、`.catch` が適用されるよりも前に例外が伝播する。テストでは `every { mock.fn() } returns flow { throw ... }` を使用する。

### MockK の default パラメータに注意

`DeviceIdService.generateGroupId(prefix, nowMillis)` の `nowMillis` はデフォルト値 `System.currentTimeMillis()`。MockK の記録時刻と実行時刻で値が変わるため、`every { ... generateGroupId(any()) }` ではマッチしない。`every { ... generateGroupId(any(), any()) }` と両引数に `any()` を指定する。

### Android Log in JVM Unit Tests

`android.util.Log.w()` は JVM ユニットテストでデフォルト例外を投げる。`android { testOptions { unitTests.isReturnDefaultValues = true } }` を `build.gradle.kts` に追加して解決。

### leaveGroup の Firestore 更新

`allowedUid` は `FieldValue.arrayRemove(uid)` で削除可能だが、`members` は複合オブジェクト配列なので `runTransaction` を避け（§12-3）、`get()` → フィルタ → `update()` で実装。

#### `SharedItem.deadline` の時刻対応

`Instant?` はすでに時刻精度を持つ。表示フォーマットを以下の通り規定した:
- 時刻 `00:00:00` → `yyyy/MM/dd`（日付のみ）
- それ以外 → `yyyy/MM/dd HH:mm`（時刻付き）

Phase 4 の期限入力ダイアログでは `DatePicker + TimePicker` を実装予定。
