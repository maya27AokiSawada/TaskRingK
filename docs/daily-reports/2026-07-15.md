## 1. 招待受諾権限エラー対応（継続）

### 背景

前日から `PERMISSION_DENIED` エラーが招待受諾時に発生。Firestore security rules が invitation update ブロック中。

### 対応（2026-07-15）

**Step 1: invitations/{token} update ルール修正**
- `firebase/firestore.rules` の invitations update ルールを段階的に簡略化
- v1: 初期状態（メンバー数＆フィールド不変チェック多数）
- v2: メンバー数チェック削除
- v3: フィールド不変チェック削除（hasAll() 削除）
- v4: 受諾時の配列更新チェック追加 → `currentUid() in request.resource.data.usedBy`

**Step 2: ルール反映完了**
```
firebase deploy --project goshopping-48db9 --only firestore:rules
→ "+  firestore: released rules firebase/firestore.rules to cloud.firestore" 確認済み
```

### 詳細コード変更

ファイル: `firebase/firestore.rules`
- `allowedJoinUpdate()` 関数をシンプル化
  - 前: 複数のフィールド検証を実施
  - 後: 最小限の条件チェックのみ（signedIn && !groupMember && currentUid in allowedUid）
- `invitations/{token}` の update ルール
  - groupMember による作成者更新を許可
  - 非メンバーによる acceptInvitation 時は usedBy 配列への追加を検証

### テスト状況

- コード層: FirestoreInvitationDataSource.acceptInvitation() で arrayUnion による usedBy 更新実装済み
- Firestore 層: usedBy フィールド検証を rules で実装済み
- デプロイ: 確認済み
- 実機テスト: Pixel でパーミッション再テスト待機中
