# 日報 2026-07-22

## 作業概要

Flutter版とのQR招待相互運用と、受諾後のUI反映不具合の解消を中心に対応。午前中の作業として、Kotlin版の招待受諾フローをFlutter版のJSON/Deep Link両対応へ寄せ、Firestore ルールも修正・反映した。

---

## 1. QR招待の相互運用対応

- Flutter版が出力する招待データ形式を確認し、Kotlin側の受け口を整理
- `InvitationPayload` / `InvitationPayloadParser` を追加し、以下を受理できるようにした
  - Flutter版の JSON 形式
  - HTTPS Deep Link 形式
  - 既存の `taskringk://invite` 形式
- Kotlin側の招待生成データも、Flutter側が必要とするキーを含むように拡張
  - `invitationId`
  - `sharedGroupId`
  - `securityKey`
  - `type`
  - `version`
- 受諾時に `securityKey` を厳格チェックするように修正

---

## 2. Firestore / 受諾フロー修正

- 招待受諾処理で、未参加ユーザーが `SharedGroups` を read しないように修正
- `allowedUid` と `members` を `arrayUnion` で更新する方式へ変更
- Firestore ルールの `SharedGroups` 読み取り条件を見直し、循環参照による `PERMISSION_DENIED` を解消
- `firebase/firestore.rules` を本番プロジェクトへデプロイ済み

---

## 3. 実機検証

- `SH-54D` を USB 接続して `prod` フレーバーを再ビルド・再インストール
- アプリ起動まで確認済み
- 途中で発生していた `permission-denied` は、Firestore ルール修正後の再デプロイで解消方向へ進めた
