# 日報 2026-07-23

## 作業概要

招待受諾後の反映遅延を、通知駆動でグループ参加処理を完結させる流れへ整理した。あわせて Android 側の warning 対応として targetSdk / 依存更新と Compose API 追従を進め、ユニットテストで回帰確認を行った。

---

## 1. 招待受諾通知フローの実装

- 招待受諾時に `INVITATION_ACCEPTED` 通知を発行し、招待者側で未読通知を監視して参加処理を実行する流れを追加
- `SharedGroupViewModel` で未読通知購読、重複実行抑止、既読化、失敗時の再実行抑止を実装
- `InvitationRepository` に通知処理経路を追加し、受諾者のグループ参加反映と `MEMBER_JOINED` 通知生成を責務分離
- 既に参加済みのグループに対する招待受諾を UI 側で事前に拒否するガードを追加

---

## 2. QR 招待データと通知マッピングの整合

- Deep Link / JSON の query decode 処理を整理し、Flutter 側と Kotlin 側の招待 payload 互換を維持
- 通知 DTO / mapper / domain model に `INVITATION_ACCEPTED` を追加し、受諾通知を end-to-end で扱えるようにした
- 招待関連 UI の文言と遷移導線を受諾通知フローに合わせて微調整

---

## 3. Android / Compose warning 対応

- `targetSdk` を 35 に更新
- `kotlinx-coroutines`、`androidx.datastore`、`google-services`、AndroidX test 系依存を更新
- `menuAnchor(MenuAnchorType.PrimaryNotEditable)` への追従、`Icons.AutoMirrored` 利用、`Locale.ROOT` 指定など lint / warning 対応を反映
- `AndroidManifest.xml` の activity label 指定を整理
